2.2文件权限操作

2.2.1文件权限的组成

一个文件或目录的权限是由三大部分组成的，分别是用户权限，用户组权限和其他组权限

每一部分权限又分为三个小部分，分别是：r(读),w(写),x(执行)，所代表的数值是r：4,w：2,x：1

如果权限为0时，使用-表示

文件权限的部分是文件描述信息中从第二位开始：如下：

第一位是文件类型：-表示文件，d表示文件夹/目录

rw-表示用户对此文件具有读``写权限

r--表示用户所在组的其他用户对此文件有读的权限

r--表示其他组的用户具有读的权限

2.2.2修改文件权限

chmod 命令

2.2.2.1字符权限

格式：chmod [ugoa]<+|-> [r|w|x]

修改a.py文件的用户权限为 rwx

为a.py文件增加其他用户组用户的执行权限

2.2.2.2数值权限

r：4

w：2

x：1

所有权限位上的数值相加为7

777：代表所用用户都具有所有权限 -rwxrwxrwx

666：代表所有用户都具有读写权限 rw-rw-rw-

400：只有用户具有读取权限 r——–

2.3服务操作

服务：后台运行的程序（没有界面），在Windows中通过win + R 输入services.msc可以打开服务管理器。在Linux中通过top命令查看正在运行的服务。这些服务提供了客户端的程序，通过客户端和服务进行交互。

2.3.1 systemctl命令

服务启动、停止、重启、启用（禁用）都需要使用systemctl命令

命令格式：

常用服务名：network(网络服务)、sshd（远程连接服务）、

start：启动服务

stop：停止服务

restrat：重启服务，由于服务的配置文件被更改，则需要重新加载配置文件

enable：启用服务（第一次使用时），启用服务之后，系统重启之后，则自动启动服务

disable：禁用服务，系统启动后不会自动启动服务

status：查看服务状态

重启网卡服务

启动ssh远程连接服务

2.3.2 srvices命令

同ststemctl一样，是管理服务，不同是命令在服务名后面。

格式：service <服务名> <start|restart|stop|status>

2.5vi编辑器

Linux强大的编辑器

2.5.1模式切换

存在三种模式：选择模式、编辑模式、命令模式

从选择模式到编辑模式：按 i 键

从编辑模式到选择模式：按 esc 键

从选择模式到命令模式：shift + :

2.5.2 编辑模式

从选择模式中：

  • 按i插入
  • 按o当前行下插入
  • 按O（shift + o) 当前行的上方插入
  • 按I移动光标到当前行的行首
  • 按A移动光标到行尾

退出并保存：先进入选择模式，按shift + :输入wq或x，如果编辑后不想保存且强制退出q!

2.5.3 命令行模式

  • /查询的内容
  • set number 显示行号
  • 行号 跳转到指定行号行首
  • w写入，不退出
  • q退出，前提没有修改文件
  • q!强制退出，不保存
  • %s/查找的内容/替换的内容/g 查找并替换全部
