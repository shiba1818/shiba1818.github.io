FastApi web框架

发表于2021-04-26|更新于2021-04-26
|阅读量:

FastApi基于WSGI（web服务网关接口）和ASGI（异步服务网关接口）两种机制，并且集成Swagger UI方便查看和调试接口。

一、安装与配置

1.1 fastapi介绍

FastApi完全按照RESTful规范，开发APP数据接口（简称API）

fastapi集成starlette(Web部分) + pydantic(数据部分) + IopenAPI(Swagger文档部分)，含包含了websocket（长链接：Web客服/即时通信IM）功能。

RESTful规范：

WSGI：Web Server Gateway Interface

Python已实现的接口

原理：分装报文，基于socket接受和发送报文

1.2 安装库

安装fastapi库

安装启动ASGI服务的库，可以使用uvicron或htpercorn

如果fastapi安装时选择[all]不用再安装uvicron

1.3 初次使用

二、请求定义

请求数据的定义，即处理函数（URI响应函数）中的参数的定义。

路径参数

必要参数，参数的值存在于URI的路径上。

如果参数具有数据类型时，请求时参数值必须符合参数类型

有效的URI资源：

  • /items/1
  • /items/55

无效的URI资源：

  • /items/abc
  • /items/1/1/2/

可以通过Enum类定义参数的预设值

可以通过Path函数来指定

【说明】Path()中的...表示必须参数，如果是None表示可选参数

查询参数

参数的位置是URI的?后面，参数在处理函数中定义同Path路径参数，但数据验证和相关的其他信息则使用fastapi.query类声明

【说明】

多个查询参数

表单参数

json数据

验证错误类型

数据验证由pydantic来完成

  • type_error.integer 数值类型不是integer类型
  • value_error.number.not_gt 数值低于指定的值
  • value_error.str.regex 正则验证
