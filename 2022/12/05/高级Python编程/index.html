装饰器 Decorator

装饰器是接受函数作为参数，添加功能后返回一个新的函数或类

装饰器原理

  1. 被装饰函数传入装饰器函数
  2. 返回内部wrapper函数引用 welcome(my_fun) => wrapper 这时候my_func变量指向的是wrapper函数my_func = wrapper
  3. 调用wrapper函数执行被装饰的函数 my_func("jack") 等于wrapper("jack")
  4. 返回被装饰函数的结果result

装饰器装饰函数后保留原来函数的信息

什么是上下文管理器？

  • context manager
  • 一个上下文管理器是一个对象
  • 定义了运行时的上下文
  • 使用 with 语句来执行
  • __enter__()安装上下文，可返回一个对象
  • __exit__()清除（释放）对象

闭包closure

函数内嵌套一个函数，内部函数调用了外部函数的变量，则会产生闭包。

__closure__会复制保存引用的变量，存储形式为元组。

单例设计模式

单例设计模式的根本目的就是要让一个类只能知道出一个实例对象。

单例的需求一般是针对需要共享的数据对象

  • 工厂对象
  • 数据库连接池对象
